# Agent-live v0.2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build local session recording, deterministic replay, adapter-oriented parsing, derived dashboard insights, and a polished README for `agent-live`.

**Architecture:** Keep the Go backend as a small single-binary app, but separate normalized events, adapters, JSONL sessions, and CLI orchestration into focused files. Keep replay and live runs flowing through the same WebSocket hub so the dashboard remains one UI. Derive insights in TypeScript from the received event list so live and replay modes share behavior.

**Tech Stack:** Go 1.26, `github.com/creack/pty`, `github.com/gorilla/websocket`, React 18, TypeScript 5.5, Vite 8, D3 force.

## Global Constraints

- Preserve existing command: `agent-live run -- <agent-command> [args...]`.
- Add `agent-live replay <session.agentlive>` without removing current behavior.
- Store recordings as local JSON Lines files only.
- Do not add hosted sharing, accounts, remote storage, authentication, or cloud sync.
- Treat missing frontend `source` and `metadata` fields as backward-compatible empty values.
- README must be updated after implementation and must not overclaim unsupported agent-specific parsing.
- Verification target: `make ci`, or equivalent `npm run build`, `npx tsc --noEmit`, and `go test ./...` with sandbox-safe Go caches.
